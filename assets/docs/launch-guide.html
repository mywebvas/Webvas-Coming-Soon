Launch Guide

Launch lean, capture real interest, stay in control.

Webvas Coming Soon Ultra-Tiny is built for site owners who need a clean pre-launch page, a usable contact list, and reliable behavior on real-world WordPress hosting without builder bloat.

Best fit: pre-launch websites, maintenance windows, soft launches, stakeholder previews, and low-friction waitlist collection.

What problem it solves

  • Hides unfinished work from visitors while trusted admins still access the real site.
  • Collects launch interest with a simple form that works with or without JavaScript.
  • Keeps launch operations clean with CSV export, source capture, and lightweight admin history.
  • Stays reliable on shared hosting, common cache layers, and mixed server environments.

What it does

  • Coming Soon and Maintenance modes with appropriate HTTP behavior.
  • Secure waitlist form with nonce checks, duplicate protection, honeypot, and rate limiting.
  • Private preview link for trusted access plus public-path allowlist rules.
  • Unique visitor counting without heavy analytics scripts or external tracking.
  • Editable brand color, heading, description, button text, and reassurance microcopy with safe defaults for fast launch setup.
  • Launch readiness signal and optional social proof powered by the same waitlist and visitor data you already collect.

Quick start

  1. Turn the page on.

    Go to Settings > Webvas Coming Soon and enable the coming soon page.

  2. Choose the right mode.

    Use Coming Soon before launch. Use Maintenance when the live site is temporarily unavailable.

  3. Set your message.

    Leave the default copy for plug-and-play speed, or update the brand color, heading, description, button text, and reassurance microcopy for your launch message.

  4. Share preview access safely.

    Use the private preview link for trusted people instead of turning the page off for everyone.

  5. Export when ready.

    Download your list as CSV, optionally filtered by date or source, with a timestamped filename that makes launch handoff cleaner.

Launch intelligence

  • The admin area shows a simple launch-readiness signal: Early, Building Momentum, or Ready to Launch.
  • The signup rate is calculated as people on the list divided by unique visitors, multiplied by 100.
  • The signal uses your waitlist count, visitor count, and signup rate instead of guesswork.
  • It is meant to guide decisions, not pretend to be a heavy analytics dashboard.

Social proof

  • Optional social proof can automatically show demand when the numbers are strong enough.
  • Low numbers stay hidden by default to avoid awkward or weak launch messaging.
  • If you prefer, you can turn it off completely or replace it with your own text.
  • Custom text can also use live placeholders like {count}, {visitors}, and {rate}.

Mode guide

Coming Soon Best before launch. Returns HTTP 200 and shows a polished launch page while you collect interest.
Maintenance Best when the live site is temporarily down. Returns HTTP 503 with retry guidance for browsers and crawlers.
Public pages Use allowlist paths only for real public pages you want open during the launch period. If a page does not exist, visitors still see the coming soon page.
Private preview Use the private preview link for clients, stakeholders, or testers who need the real site without exposing it publicly.

Security and privacy

  • No external tracking scripts or third-party analytics calls.
  • Data stays in your WordPress database.
  • Form submissions use nonce validation, safe redirects, and signed tracking context.
  • Only trusted plugin managers can open settings, export contacts, or change plugin state by default.

Performance and scaling truth

  • The frontend is intentionally lightweight and avoids theme and builder overhead.
  • The page is also intentionally dynamic so preview access, honest visitor counts, and form behavior remain reliable.
  • That means this plugin is optimized for real launches, not for pretending a dynamic page is fully static at the edge.
If you expect extremely large anonymous spikes, the right next step is a dedicated static-edge mode, not extra complexity inside the default launch flow.

Best practices

  • Keep the headline short and strong. Use the description to explain the benefit of joining the list, and use the short text under the button to remove fear.
  • Match the brand color to the site you are about to launch so the page feels intentional, not generic.
  • Use only simple inline emphasis in the description, such as <strong>, <em>, <br>, or a safe link.
  • Share the preview link only with people who genuinely need access before launch.
  • Use UTM parameters on campaigns so CSV exports stay actionable at launch time.
  • After launch, export your contacts, follow up quickly, then clear saved data if you no longer need it.
